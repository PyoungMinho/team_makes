# 반려동물 보호자를 위한 앱 기획 최종안

## 결론 요약

창의팀의 "반려동물 건강 안심 플랫폼" 비전은 시장의 실질적 페인포인트를 정확히 짚었으나, 비평팀이 지적한 기능 과부하, 초기 자본 부담, 의료 법적 리스크는 무시할 수 없는 현실이다. 따라서 **비평팀이 제안한 "의료기록 + 보험 청구 자동화"를 1차 진입점으로 채택**하되, 창의팀의 응급처치 챗봇을 **트리아지(분류) 도구로 한정하여 초기부터 병행 탑재**한다. 이유는 단순하다. 의료기록 디지털화만으로는 사용자가 앱을 깔아야 할 즉각적 동기가 약하고, 응급 상황이라는 감정적 트리거가 있어야 설치 전환율이 생긴다. 단, AI 행동 분석기와 유전자 검사 등 검증되지 않은 고비용 기능은 데이터가 충분히 쌓인 Phase 3 이후로 미룬다.

---

## 채택한 아이디어

### 1. 의료기록 전자수첩 + 보험 청구 자동화 (핵심 축)
- **출처**: 창의팀 제안 3번 + 비평팀 대안의 공통 교집합
- **채택 이유**: 양측 모두 이 기능의 시장성을 가장 높게 평가했다. 비평팀조차 "가장 강력한 아이디어"로 인정했으며, 보호자의 반복적 불편(종이 수첩, 수기 보험 청구)을 해결하는 명확한 가치 제안이 존재한다. 병원 파트너십을 통한 자연스러운 유입 채널 확보가 가능하고, 데이터 축적 자체가 후속 기능의 연료가 된다.

### 2. 응급처치 트리아지 챗봇 (초기 견인용)
- **출처**: 창의팀 제안 1번
- **채택 이유**: 비평팀도 "정서적 가치"를 인정한 기능이다. 핵심은 **진단이 아닌 분류**다. "지금 응급실에 가야 하는가, 내일 가도 되는가"만 판단하고 가까운 야간 병원을 안내한다. 이 기능은 (1) 앱 설치의 감정적 동기를 만들고, (2) 파트너 병원으로의 환자 유입 채널이 되며, (3) 의료기록 기능과 자연스럽게 연결된다. 법적 리스크는 "수의사 판단을 대체하지 않는 참고 정보" 면책 고지로 관리한다.

### 3. 보류: AI 행동 분석기
- **출처**: 창의팀 제안 2번
- **판단**: Phase 3 이후 데이터 기반 도입 검토. 현 시점에서는 행동 데이터 수집 방법(웨어러블? 카메라? 수동 입력?)이 불명확하고, 정확도 검증에 필요한 임상 데이터가 없다. 의료기록 데이터가 수만 건 쌓인 후 역학 패턴 분석으로 접근하는 것이 현실적이다.

## 반영한 리스크 대응

| 비평팀 지적 | 반영 방식 |
|---|---|
| **기능 과부하 = 전략 부재** | 핵심 기능을 2개(의료기록 + 트리아지 챗봇)로 축소. SNS, 쇼핑, 피트니스 등 부수 기능은 전부 제거. |
| **단위 경제학 미검증** | Phase 1에서 무료 모델로 시작하여 실제 CAC/리텐션 데이터 확보 후 과금 전환. 초기부터 구독 모델을 걸지 않는다. |
| **초기 자본 20억+ 부담** | MVP 범위를 의료기록 입력/조회 + 챗봇 룰베이스 엔진으로 최소화. 초기 개발비를 3-5억 수준으로 억제. |
| **수의사 파트너십 18-24개월** | Phase 0에서 파트너 병원 20-30곳만 확보하는 것을 목표로 축소. 전국 커버리지가 아닌 수도권 집중. |
| **AI 오진 법적 리스크** | 챗봇을 "진단"이 아닌 "트리아지(분류)"로 명확히 포지셔닝. 모든 응답에 면책 고지 삽입. 수의사 자문위원회 운영으로 컨텐츠 검수. |
| **CAC 폭발** | 병원 내 QR코드 + 수의사 추천을 1차 채널로 설정. 퍼포먼스 마케팅은 Phase 2 이후 검증된 단위 경제 확보 후 집행. |
| **보험사 제휴 난이도** | Phase 1에서는 보험 "청구 서류 자동 생성"까지만 구현. 보험사 API 직접 연동은 Phase 2에서 1개 보험사 파일럿으로 시작. |

## 실행 계획

### Phase 0: 검증 (0~3개월) — 투자 규모: 1.5억

| 항목 | 내용 |
|---|---|
| **목표** | 핵심 가설 2개 검증: (1) 보호자가 의료기록을 디지털로 관리할 의향이 있는가, (2) 응급 상황에서 챗봇을 신뢰하는가 |
| **산출물** | 의료기록 입력/조회 + 트리아지 챗봇 MVP (iOS 단일 플랫폼) |
| **파트너십** | 수도권 동물병원 10곳 MOU, 수의사 자문위원 3인 확보 |
| **사용자** | 클로즈드 베타 300명 (파트너 병원 내원 고객 대상) |
| **핵심 지표** | 주간 활성 사용률 40% 이상, 의료기록 등록 건수 월 500건 이상, 챗봇 만족도 NPS 30 이상 |
| **Go/No-Go** | 주간 활성 30% 미만 시 피봇 또는 중단 결정 |

### Phase 1: 제품-시장 적합성 (4~9개월) — 투자 규모: 3억 추가

| 항목 | 내용 |
|---|---|
| **목표** | PMF 달성 + 단위 경제 구조 검증 |
| **기능 확장** | 예방접종 알림, 보험 청구 서류 자동 생성, Android 출시 |
| **파트너십** | 파트너 병원 30곳 확대, 보험사 1곳 데이터 연동 파일럿 |
| **사용자** | MAU 5,000명 목표 |
| **수익 모델 테스트** | 프리미엄 구독(월 3,900원) 도입 — 보험 서류 자동화 + 무제한 챗봇 이용. 무료 티어 유지(월 챗봇 3회 + 기본 기록). |
| **핵심 지표** | 유료 전환율 5% 이상, 월 이탈률 8% 이하, CAC < LTV/3 |

### Phase 2: 성장 (10~18개월) — 투자 규모: 시리즈 A 유치 목표

| 항목 | 내용 |
|---|---|
| **목표** | MAU 50,000명, 월 매출 1억 원 |
| **기능 확장** | 병원 EMR 시스템 연동 (상위 EMR 2개사), 보험사 API 직접 연동(청구 원클릭), AI 트리아지 고도화(이미지 입력 지원) |
| **파트너십** | 파트너 병원 200곳, 보험사 2-3곳 |
| **수익 다각화** | 구독 + 보험 중개 수수료 + 병원 프리미엄 리스팅 |
| **마케팅** | 검증된 단위 경제 기반 퍼포먼스 마케팅 시작 |

### Phase 3: 확장 (19개월~)

| 항목 | 내용 |
|---|---|
| **데이터 사업** | 익명화된 의료 데이터 기반 B2B 인사이트 판매 (사료/제약사 대상) |
| **AI 고도화 검토** | 축적된 의료기록 데이터 기반 행동 분석, 질병 예측 모델 개발 타당성 평가 |
| **해외 진출** | 일본 시장 우선 검토 (반려동물 시장 규모 + 디지털 수용도) |

## 보류/추가 검토 사항

### 1. AI 행동 분석기 도입 시점과 방식
- 창의팀이 제안한 행동 분석 기능은 기술적 매력이 있으나, 데이터 수집 방법과 정확도 검증 문제가 해결되지 않았다. Phase 3에서 축적된 의료기록 데이터를 기반으로 재검토한다. 웨어러블 디바이스 연동 vs 카메라 기반 vs 보호자 수동 입력 중 어떤 방식이 현실적인지 별도 리서치가 필요하다.

### 2. 수의사 법적 자문 체계
- 트리아지 챗봇의 법적 경계선(수의사법 위반 여부)에 대해 법률 자문을 Phase 0 시작 전에 반드시 완료해야 한다. "정보 제공"과 "진료 행위"의 경계에 대한 명확한 법적 의견서가 필요하다.

### 3. 보험사 제휴 구조의 구체화
- 보험 중개 수수료 모델이 보험업법상 가능한 구조인지, 보험대리점 등록이 필요한지 법률 검토가 선행되어야 한다. Phase 1 시작 전까지 결론을 내린다.

### 4. 데이터 프라이버시 및 개인정보 보호
- 반려동물 의료 데이터가 개인정보보호법 적용 대상인지, B2B 판매 시 어떤 수준의 익명화가 필요한지 법적 검토가 필요하다. 보호자의 개인정보(이름, 연락처)와 반려동물 정보의 결합 시 민감정보 해당 여부를 확인한다.

### 5. 경쟁사 대응 시나리오
- 기존 반려동물 앱(포인핸드, 핏펫 등)이 유사 기능을 출시할 경우의 방어 전략이 구체화되지 않았다. 특히 병원 EMR 연동에서의 독점 계약 가능성, 보험사 제휴에서의 선점 효과 등을 Phase 1 기간 내 전략화해야 한다.

## 의사결정 근거 요약

| 판단 기준 | 창의팀 원안 | 비평팀 대안 | 최종안 |
|---|---|---|---|
| 초기 범위 | 3개 기능 동시 | 1개 기능 집중 | **2개 기능 (기록+챗봇)** |
| 첫 수익 시점 | Phase 1 (4개월) | Phase 2 (10개월) | **Phase 1 (4개월, 테스트)** |
| 초기 투자 | 추정 10억+ | 추정 3-5억 | **4.5억 (Phase 0+1)** |
| 사용자 견인력 | 높음 (다양한 기능) | 낮음 (실용적이나 지루함) | **중상 (실용+감정적 트리거)** |
| 리스크 수준 | 높음 | 낮음 | **중간** |

창의팀의 비전이 방향을 잡았고, 비평팀의 현실 감각이 범위를 깎았다. 최종안은 양측의 강점을 취하되, **"작게 시작하고 데이터로 증명한 후 넓히는"** 원칙을 따른다.
