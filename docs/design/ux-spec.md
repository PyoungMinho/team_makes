# UX 설계서 — 반려동물 건강 안심 플랫폼

> 작성: UX 디자이너 / 2026-04-13
> 기준 문서: project-direction.md (팀장 합동 회의 확정안, 2026-04-11)
> 대상 플랫폼: iOS (React Native / Expo) — Phase 0 MVP

## 1. 사용자 페르소나

### 페르소나 A — 주요 타겟: "꼼꼼한 보호자" 박지은 (32세)

| 항목 | 내용 |
|---|---|
| 직업 | 마케터 (스타트업 재직) |
| 반려동물 | 말티즈 2마리 (4세, 2세) |
| 기기 환경 | iPhone 14, 앱 평균 사용 시간 3-4시간/일 |
| 디지털 친숙도 | 높음 — 새 앱 도입 저항 낮음 |

**핵심 페인포인트**
- 두 마리의 접종, 심장사상충, 미용 일정이 뒤섞여 수첩에 기록하다 놓침
- 야간 응급 상황(새벽 2시 구토)에 어디로 가야 할지 몰라 공황 상태 경험
- 보험 청구 시 영수증을 사진 찍어 카카오톡으로 제출하는 과정이 번거로움

**목표**
- 두 마리의 의료 이력을 한 곳에서 빠르게 비교·조회
- 응급 상황에서 "지금 병원 가야 해?" 판단을 빠르게 받고 싶음

**사용 시나리오**
- 병원 방문 후 진료 영수증을 즉시 촬영해 기록 저장
- 응급 챗봇으로 증상 체크 후 야간 병원 주소 공유

---

### 페르소나 B — 서브 타겟: "처음 키우는 집사" 이현준 (27세)

| 항목 | 내용 |
|---|---|
| 직업 | 대학원생 |
| 반려동물 | 고양이 1마리 (7개월, 입양 3개월 차) |
| 기기 환경 | iPhone SE (3세대), 배터리 성능 우려 |
| 디지털 친숙도 | 중간 — 복잡한 UI에 이탈률 높음 |

**핵심 페인포인트**
- 어떤 검사를 언제 해야 하는지 모름 (예방접종 스케줄 무지)
- 고양이 증상이 심각한지 일반적인지 판단 기준이 없음
- 의료기록이 없어 새 병원 방문 시 이전 이력 공유 불가

**목표**
- 간단하게 기록을 쌓아 나가고 싶음 (복잡한 입력 폼 거부감)
- 증상이 생기면 즉각적인 가이드가 필요

**사용 시나리오**
- 고양이 기침 증상 발생 시 트리아지 챗봇으로 즉각 상담
- 예방접종 알림을 받고 기록 한 줄 추가

## 2. 핵심 사용자 플로우 5개

### Flow 1 — 온보딩 (첫 가입 → 펫 등록)

```
[앱 최초 실행]
      |
      v
[스플래시 + 브랜드 로고] (1.5초)
      |
      v
[면책 동의 화면]
  - "본 앱은 수의사 진료를 대체하지 않습니다" 안내
  - [동의하고 시작] 버튼 (단일 CTA)
      |
      v
[소셜 로그인 / 이메일 가입] (Step 1/4)
  - Apple ID (iOS 우선, App Store 정책 필수)
  - 카카오 로그인
  - 이메일 직접 가입
      |
      v
[보호자 이름 입력] (Step 2/4)
  - 닉네임 1개 필드
  - [건너뛰기] 없음 — 필수
      |
      v
[첫 펫 등록] (Step 3/4)
  - 펫 이름 (필수)
  - 종 선택: 강아지 / 고양이 / 기타 (세그먼트 컨트롤)
  - 품종 입력 (선택, 검색 자동완성)
  - 생년월일 또는 나이 (선택)
  - 성별 (선택)
  - 대표 사진 (선택 — 카메라 / 갤러리)
  - [등록하기] 버튼
      |
      v
[권한 요청] (Step 4/4)
  - 카메라 권한: "영수증 OCR 촬영에 필요합니다"
  - 알림 권한: "접종 일정 리마인더에 필요합니다"
  - 위치 권한: "응급 시 근처 야간 병원 안내에 필요합니다"
  - 각 권한: [허용] / [나중에] 선택 가능
      |
      v
[웰컴 화면] — "지은님과 초코의 건강 여정을 시작해요"
  - 핵심 기능 3개 스와이프 카드 (의료기록 / 응급 챗봇 / 알림)
  - [시작하기] → 내 아이 탭 메인으로 진입
```

**에러 상태**
- 소셜 로그인 실패: "로그인에 실패했습니다. 다시 시도해주세요" + 재시도 버튼
- 네트워크 없음: 오프라인 배너 노출, 입력값 로컬 임시 저장 후 재연결 시 자동 전송

### Flow 2 — 의료기록 추가 (OCR 촬영 → 저장)

```
[내 아이 탭 - 펫 상세 화면]
      |
      v
[+ 기록 추가] FAB 버튼 탭
      |
      v
[입력 방식 선택 바텀시트]
  - [영수증 촬영 (권장)] — OCR 아이콘
  - [직접 입력] — 편집 아이콘
      |
      +--- OCR 선택 ---+--- 직접 입력 선택 ---+
      |                                      |
      v                                      v
[카메라 뷰파인더]                        [수동 입력 폼]
  - 가이드 프레임 오버레이                  (Flow 2-B 참고)
  - "영수증을 프레임 안에"
  - [촬영] 버튼
      |
      v
[촬영 결과 미리보기]
  - 재촬영 / 사용 선택
      |
      v
[OCR 분석 중] — 로딩 애니메이션
  - "영수증을 읽고 있어요..." (3-8초)
      |
      v
[OCR 결과 확인 화면]
  - 자동 추출 필드 표시 (하이라이트)
    · 병원명, 진료일, 진료 내용, 금액
  - 인식률 낮은 필드: 주황색 테두리 + "확인 필요"
  - 사용자 수정 가능 (인라인 편집)
  - 추가 입력 필드:
    · 메모 (선택)
    · 진단명 (선택)
    · 처방약 (선택)
      |
      v
[저장 확인]
  - [저장하기] 버튼
      |
      v
[저장 완료 피드백]
  - 토스트 메시지: "기록이 저장되었어요"
  - 타임라인에 새 항목 추가됨 (애니메이션)
```

**Flow 2-B — 직접 입력**
```
[수동 입력 폼]
  - 병원명 (텍스트, 필수)
  - 진료일 (날짜 피커, 필수)
  - 진료 유형 (칩 선택: 정기검진 / 예방접종 / 응급 / 수술 / 기타)
  - 진료 내용 (텍스트, 선택)
  - 금액 (숫자, 선택)
  - 진단명 (텍스트, 선택)
  - 처방약 (텍스트, 선택)
  - 첨부 파일 (갤러리 선택, 선택)
  - [저장하기] 버튼
```

**에러 상태**
- OCR 인식 실패 (인식률 0%): "영수증을 인식하지 못했어요. 직접 입력으로 전환할까요?" → [직접 입력] / [재촬영]
- 업로드 실패: "저장에 실패했습니다. 데이터는 임시 보관되었습니다" + 재시도
- 카메라 권한 없음: "기록 촬영을 위해 카메라 권한이 필요합니다" + 설정으로 이동 딥링크

### Flow 3 — 의료기록 조회

```
[내 아이 탭 - 펫 상세 화면]
  - 상단: 펫 프로필 카드 (이름, 품종, 나이)
  - 하단: 의료기록 타임라인 (최신순)
      |
      v
[타임라인 항목 탭]
      |
      v
[기록 상세 화면]
  - 병원명, 진료일, 진료 유형 칩
  - 진료 내용 / 진단명 / 처방약
  - 금액
  - 첨부 이미지 썸네일 (탭 시 전체화면)
  - [수정] / [삭제] 액션
      |
      v (조회 완료 — 단방향 플로우)
```

**필터 및 검색 (타임라인 내)**
- 진료 유형 필터 탭 (전체 / 정기검진 / 예방접종 / 응급 / 수술)
- 기간 필터 (최근 3개월 / 6개월 / 1년 / 전체)

**오프라인 동작**
- 로컬 SQLite 캐시에서 기록 조회 가능
- 오프라인 배너 ("오프라인 모드 — 최근 동기화 기준으로 표시합니다") 상단 고정

**빈 상태**
- 일러스트 + "아직 기록이 없어요. 첫 기록을 추가해볼까요?" + [+ 기록 추가] 버튼

---

### Flow 4 — 트리아지 챗봇 사용

```
[응급 탭 진입]
      |
      v
[챗봇 면책 배너] (상단 고정, 항상 노출)
  - "이 정보는 수의사 진료를 대체하지 않습니다"
      |
      v
[챗봇 홈 화면]
  - 어떤 아이 증상인지 선택 (멀티펫의 경우)
    · 펫 선택 칩 (이름 + 아이콘)
  - 증상 빠른선택 카드 8개 (2열 그리드)
    · 구토/구역 / 설사/혈변 / 외상/출혈 / 호흡 이상
    · 경련/발작 / 먹지 않음 / 음수 이상 / 기타
      |
      +--- 빠른선택 카드 탭 ---+--- 텍스트 직접 입력 ---+
      |                                               |
      v                                               v
[챗봇 대화 화면]                               [챗봇 대화 화면]
  - 선택한 증상 → 첫 메시지로 자동 전송          - 사용자가 자유 입력
      |
      v
[AI 응답 스트리밍]
  - SSE 스트리밍, 타이핑 인디케이터 (점 3개 애니메이션)
  - 응답 완료 후 구조화 결과 카드:
    · 응급도 배지: 즉시 / 오늘 내 / 내일 가능 / 관찰 필요
    · 주요 체크 리스트 (2-4개 항목)
    · 권장 행동 요약
      |
      v
[트리아지 결과]
  - 응급도 "즉시" 판정 시:
    · 빨간 배너 + "지금 응급실로 이동하세요"
    · [근처 야간 병원 찾기] 버튼 → 지도 화면 푸시
  - 응급도 "오늘 내" / "내일 가능":
    · 노란/초록 배너
    · 홈케어 가이드 카드
    · [병원 찾기] 버튼 (선택적)
      |
      v
[대화 계속 또는 종료]
  - [다른 증상 추가]
  - [대화 저장] (의료기록에 자동 연결 저장 — 선택)
  - [탭 이동] 하단 네비게이션으로 이탈
```

**에러 상태**
- 네트워크 없음: "챗봇은 인터넷 연결이 필요합니다. 의료기록은 오프라인에서도 확인 가능합니다"
- LLM 응답 타임아웃 (>10초): "응답이 지연되고 있어요. 잠시 후 다시 시도해주세요" + 재시도 버튼
- 월 무료 쿼리 소진: "이번 달 무료 상담 3회를 모두 사용했어요" + 프리미엄 업그레이드 안내 (Phase 1 대비)

### Flow 5 — 펫 전환 (멀티펫)

```
[내 아이 탭 - 어떤 화면에서든]
      |
      v
[상단 펫 선택 바]
  - 현재 선택된 펫 이름 + 아바타 (좌측)
  - 펫 아바타 행 (수평 스크롤, 최대 5마리 표시)
  - [+ 추가] 원형 버튼 (마지막 위치)
      |
      +--- 다른 펫 아바타 탭 ---+--- + 추가 탭 ---+
      |                                           |
      v                                           v
[선택된 펫 데이터로 화면 전환]             [펫 추가 플로우]
  - 크로스페이드 트랜지션 (200ms)           (온보딩 Step 3과 동일 폼)
  - 타임라인, 기록 모두 전환
```

**응급 탭에서의 펫 전환**
- 챗봇 대화 시작 전 "어떤 아이를 위한 상담인가요?" 단계에서 펫 선택
- 대화 중 전환 불가 (UX 혼선 방지) — 전환 시 "현재 대화를 종료하고 전환하시겠어요?" 확인 모달

## 3. 정보 구조 (IA)

```
반려동물 건강 안심 플랫폼
├── 온보딩 플로우 (비인증)
│   ├── 스플래시
│   ├── 면책 동의
│   ├── 소셜/이메일 로그인
│   ├── 보호자 이름 입력
│   ├── 첫 펫 등록
│   ├── 권한 요청
│   └── 웰컴
│
├── [탭 1] 내 아이
│   ├── 메인 (펫 선택 바 + 현재 펫 요약 카드)
│   └── 펫 상세
│       ├── 기본 정보 (이름, 품종, 나이, 성별, 사진)
│       ├── 의료기록 타임라인
│       │   ├── 기록 상세 (조회 / 수정 / 삭제)
│       │   └── 기록 추가
│       │       ├── OCR 촬영 플로우
│       │       │   ├── 카메라 뷰파인더
│       │       │   ├── OCR 결과 확인 및 수정
│       │       │   └── 저장 완료
│       │       └── 수동 입력 폼
│       └── 펫 정보 편집
│
├── [탭 2] 응급
│   ├── 챗봇 홈 (증상 빠른선택 카드)
│   ├── 챗봇 대화 화면
│   │   ├── AI 스트리밍 응답
│   │   └── 트리아지 결과 카드
│   └── 근처 야간 병원 지도
│       └── 병원 상세 (전화, 주소, 영업시간)
│
└── [탭 3] 더보기
    ├── 보호자 프로필
    │   └── 계정 정보 편집
    ├── 알림 설정
    │   ├── 접종 리마인더 ON/OFF
    │   └── 챗봇 알림 ON/OFF
    ├── 앱 정보
    │   ├── 버전 정보
    │   ├── 이용약관
    │   ├── 개인정보 처리방침
    │   └── 면책 고지문
    └── 로그아웃 / 탈퇴
```

**탭 뱃지 규칙**
- 내 아이 탭: 다가오는 접종 예정일 N일 이내 시 숫자 뱃지 (Phase 1)
- 응급 탭: 뱃지 없음 — 항상 즉시 접근 가능해야 함
- 더보기 탭: 읽지 않은 공지사항 있을 시 빨간 점 뱃지

## 5. 인터랙션 노트

### 5-1. 트랜지션 & 애니메이션

| 인터랙션 | 방식 | 지속 시간 | 비고 |
|---|---|---|---|
| 탭 전환 | 즉시 전환 (애니메이션 없음) | 0ms | 응급 탭 접근성 최우선 |
| 스택 푸시 (내 아이 탭) | 슬라이드 왼쪽 (iOS 기본) | 300ms | RN NavigationStack 기본 |
| 펫 전환 (선택 바) | 크로스페이드 | 200ms | 데이터 교체 체감 완화 |
| OCR 결과 로드 | 스켈레톤 UI → 컨텐츠 | 필드별 순차 표시 | 인식 완료 필드부터 순서대로 |
| 트리아지 결과 카드 | 슬라이드업 | 350ms | ease-out 커브 |
| 타임라인 새 항목 추가 | 페이드인 + 슬라이드다운 | 400ms | 저장 완료 직후 |
| 로딩 스피너 | 원형 인디케이터 | 무한 반복 | 전체 화면 블로킹 지양 |

### 5-2. 제스처

| 화면 | 제스처 | 동작 |
|---|---|---|
| 펫 상세 타임라인 | 스와이프 왼쪽 (카드) | 삭제 버튼 노출 |
| 챗봇 대화 화면 | 스와이프 아래 (키보드) | 키보드 dismiss |
| OCR 결과 이미지 썸네일 | 탭 | 전체 화면 이미지 뷰어 모달 |
| 전체 화면 이미지 뷰어 | 핀치 줌 | 확대/축소 |
| 전체 화면 이미지 뷰어 | 스와이프 아래 | 모달 닫기 |
| 펫 선택 바 | 수평 스크롤 | 펫 목록 탐색 |

### 5-3. 햅틱 피드백

| 트리거 | 햅틱 유형 | 근거 |
|---|---|---|
| OCR 촬영 버튼 | Impact (Medium) | 셔터 느낌 전달 |
| 저장 완료 | Notification (Success) | 긍정 피드백 |
| 에러/경고 | Notification (Error) | 주의 환기 |
| 응급도 "즉시" 결과 카드 등장 | Impact (Heavy) | 긴박감 전달 |
| 펫 전환 | Impact (Light) | 전환 확인 |

### 5-4. 접근성 (WCAG 2.1 AA 기준)

| 항목 | 적용 방식 |
|---|---|
| 색상 대비 | 텍스트-배경 대비 4.5:1 이상, 큰 텍스트 3:1 이상 |
| 터치 타겟 크기 | 최소 44x44pt (iOS HIG 기준) |
| VoiceOver | 모든 인터랙티브 요소 accessibilityLabel 필수 제공 |
| 응급도 배지 | 색상에만 의존하지 않음 — 텍스트 레이블 병기 ("즉시", "오늘 내") |
| 폰트 크기 | 시스템 Dynamic Type 지원, 최소 14px 이상 |
| 애니메이션 | iOS 동작 줄이기(Reduce Motion) 설정 시 트랜지션 즉시 전환으로 대체 |

### 5-5. 응급 탭 특별 UX 규칙

응급 탭은 일반 UX 원칙보다 **긴박성 우선** 원칙이 적용된다.

1. **탭 1회 진입 원칙**: 응급 탭 아이콘 탭 → 즉시 증상 선택 화면 도달 (중간 단계 없음)
2. **영구 면책 배너**: 스크롤 시 헤더와 함께 고정, 접기 불가
3. **색상 신호 강화**: 응급도 배지는 색상 + 텍스트 + 아이콘 3중 신호
4. **결과 저장 옵션 제공**: 대화 내용을 의료기록에 연결 저장 가능 (선택)
5. **야간 모드 강제 미사용**: 응급 상황은 밝은 모드에서 더 가독성 높음 — 시스템 다크모드 무시 옵션 검토

### 5-6. 멀티펫 UX 일관성 규칙

1. **전역 펫 컨텍스트**: 어떤 화면에서도 현재 선택된 펫이 상단에 표시됨
2. **응급 탭 펫 선택**: 대화 시작 전 선택 — 대화 중 전환 시 확인 모달 필수
3. **기록 추가 귀속**: FAB로 기록 추가 시 현재 선택된 펫에 자동 귀속
4. **펫 추가 진입점**: 펫 선택 바의 [+] 버튼 — 더보기 탭에는 노출 안 함 (단일 진입점)
